# Locations API

This guide covers the API endpoints for managing locations in the Revenue Recognition System. Locations represent physical places such as offices, warehouses, or stores where business activities occur. {{ className: 'lead' }}

## Overview

The Locations API allows you to:
- Create new locations
- Retrieve location information
- Update existing locations
- Delete locations
- List locations with filtering and pagination

All location operations require organization context and authentication.

## Schema

| Field          | Type      | Description                                           | Required |
|----------------|-----------|-------------------------------------------------------|---------| 
| id             | UUID      | Unique identifier (system-generated)                  | Read-only|
| organizationId | UUID      | Organization this location belongs to                 | Read-only|
| subsidiaryId   | UUID      | Subsidiary this location belongs to                   | Yes     |
| name           | String    | Name of the location                                  | Yes     |
| code           | String    | Unique code for the location (within organization)    | No      |
| description    | String    | Optional description                                  | No      |
| addressLine1   | String    | First line of address                                 | No      |
| addressLine2   | String    | Second line of address                                | No      |
| city           | String    | City                                                  | No      |
| stateProvince  | String    | State or province                                     | No      |
| postalCode     | String    | Postal or ZIP code                                    | No      |
| countryCode    | String    | ISO 2-letter country code (e.g., "US")                | No      |
| isActive       | Boolean   | Whether the location is active (default: true)        | No      |
| createdAt      | DateTime  | When the record was created                           | Read-only|
| updatedAt      | DateTime  | When the record was last updated                      | Read-only|

## Error Handling

| Status Code | Description                                            |
|-------------|--------------------------------------------------------|
| 400         | Bad Request - Invalid input data                       |
| 401         | Unauthorized - Missing or invalid authentication token |
| 404         | Not Found - Location with the specified ID not found   |
| 409         | Conflict - Resource already exists with the same code  |
| 500         | Internal Server Error                                  |

## Code Examples

### JavaScript/TypeScript

```javascript
// Create a new location
const createLocation = async () => {
  const response = await fetch('/api/v1/locations', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + token,
      'x-stytch-organization-id': stytchOrgId
    },
    body: JSON.stringify({
      subsidiaryId: 'f67f0ea2-77bd-4578-8c76-0116d473cc27',
      name: 'New York Office',
      code: 'NYC',
      addressLine1: '456 Broadway',
      city: 'New York',
      stateProvince: 'NY',
      postalCode: '10013',
      countryCode: 'US'
    })
  });
  
  return await response.json();
};

// Get locations by country code
const getLocationsByCountry = async (countryCode) => {
  const response = await fetch(`/api/v1/locations?countryCode=${countryCode}`, {
    headers: {
      'Authorization': 'Bearer ' + token,
      'x-stytch-organization-id': stytchOrgId
    }
  });
  
  return await response.json();
};
```
