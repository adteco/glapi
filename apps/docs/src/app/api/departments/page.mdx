# Departments API

This guide covers the API endpoints for managing departments in the Revenue Recognition System. Departments are organizational units used for categorizing and tracking costs, revenue, and other accounting data within a subsidiary. {{ className: 'lead' }}

## Overview

The Departments API allows you to:
- Create new departments
- Retrieve department information
- Update existing departments
- Delete departments
- List departments with filtering and pagination

All department operations require organization context and authentication.

## Schema

| Field          | Type      | Description                                           | Required |
|----------------|-----------|-------------------------------------------------------|---------| 
| id             | UUID      | Unique identifier (system-generated)                  | Read-only|
| organizationId | UUID      | Organization this department belongs to               | Read-only|
| subsidiaryId   | UUID      | Subsidiary this department belongs to                 | Yes     |
| name           | String    | Name of the department                                | Yes     |
| code           | String    | Unique code for the department (within organization)  | No      |
| description    | String    | Optional description                                  | No      |
| isActive       | Boolean   | Whether the department is active (default: true)      | No      |
| createdAt      | DateTime  | When the record was created                           | Read-only|
| updatedAt      | DateTime  | When the record was last updated                      | Read-only|

### List Departments

#### Query Parameters

| Parameter      | Type    | Description                                           | Default |
|----------------|---------|-------------------------------------------------------|---------| 
| page           | Integer | Page number for pagination                            | 1       |
| limit          | Integer | Number of items per page                              | 10      |
| sortField      | String  | Field to sort by (name, code, createdAt)              | name    |
| sortOrder      | String  | Sort direction (asc, desc)                            | asc     |
| subsidiaryId   | UUID    | Filter by subsidiary ID                               | -       |

## Error Handling

| Status Code | Description                                              |
|-------------|----------------------------------------------------------|
| 400         | Bad Request - Invalid input data                         |
| 401         | Unauthorized - Missing or invalid authentication token   |
| 404         | Not Found - Department with the specified ID not found   |
| 409         | Conflict - Resource already exists with the same code    |
| 500         | Internal Server Error                                    |

## Code Examples

### JavaScript/TypeScript

```javascript
// Create a new department
const createDepartment = async () => {
  const response = await fetch('/api/v1/departments', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + token,
      'x-stytch-organization-id': stytchOrgId
    },
    body: JSON.stringify({
      subsidiaryId: 'f67f0ea2-77bd-4578-8c76-0116d473cc27',
      name: 'Operations',
      code: 'OPS',
      description: 'Operations department'
    })
  });
  
  return await response.json();
};

// Get departments by subsidiary
const getDepartmentsBySubsidiary = async (subsidiaryId) => {
  const response = await fetch(`/api/v1/departments?subsidiaryId=${subsidiaryId}`, {
    headers: {
      'Authorization': 'Bearer ' + token,
      'x-stytch-organization-id': stytchOrgId
    }
  });
  
  return await response.json();
};
```
