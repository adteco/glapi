# Classes API

This guide covers the API endpoints for managing classes in the Revenue Recognition System. Classes are accounting dimensions used for categorizing transactions by business purpose or segment. {{ className: 'lead' }}

## Overview

The Classes API allows you to:
- Create new classes
- Retrieve class information
- Update existing classes
- Delete classes
- List classes with filtering and pagination

All class operations require organization context and authentication.

## Schema

| Field          | Type      | Description                                           | Required |
|----------------|-----------|-------------------------------------------------------|---------| 
| id             | UUID      | Unique identifier (system-generated)                  | Read-only|
| organizationId | UUID      | Organization this class belongs to                    | Read-only|
| subsidiaryId   | UUID      | Subsidiary this class belongs to                      | Yes     |
| name           | String    | Name of the class                                     | Yes     |
| code           | String    | Unique code for the class (within organization)       | No      |
| description    | String    | Optional description                                  | No      |
| isActive       | Boolean   | Whether the class is active (default: true)           | No      |
| createdAt      | DateTime  | When the record was created                           | Read-only|
| updatedAt      | DateTime  | When the record was last updated                      | Read-only|

|----------------|---------|-------------------------------------------------------|---------|

## Error Handling

| Status Code | Description                                            |
|-------------|--------------------------------------------------------|
| 400         | Bad Request - Invalid input data                       |
| 401         | Unauthorized - Missing or invalid authentication token |
| 404         | Not Found - Class with the specified ID not found      |
| 409         | Conflict - Resource already exists with the same code  |
| 500         | Internal Server Error                                  |

## Code Examples

### JavaScript/TypeScript

```javascript
// Create a new class
const createClass = async () => {
  const response = await fetch('/api/v1/classes', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + token,
      'x-stytch-organization-id': stytchOrgId
    },
    body: JSON.stringify({
      subsidiaryId: 'f67f0ea2-77bd-4578-8c76-0116d473cc27',
      name: 'E-commerce',
      code: 'ECOM',
      description: 'E-commerce business segment'
    })
  });
  
  return await response.json();
};

// Get classes by subsidiary
const getClassesBySubsidiary = async (subsidiaryId) => {
  const response = await fetch(`/api/v1/classes?subsidiaryId=${subsidiaryId}`, {
    headers: {
      'Authorization': 'Bearer ' + token,
      'x-stytch-organization-id': stytchOrgId
    }
  });
  
  return await response.json();
};
```
